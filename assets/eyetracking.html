How to add eyetracking with markers to your exsisiting Psychopy experiment

Before we start, make sure you have the following installed:

  • Psychopy Preferably the latest version.
  • tobii_research Tobii Pro SDK for Python. Install it in your psychopy environment. If you don't know how to do it, follow the instructions below.

For PsychoPy Standalone Version

Open Coder View: Launch PsychoPy and switch to the Coder view. You can do this by clicking on the "View" menu and selecting "Open Coder view" or by clicking on the "Coder" button if available.

Windows: Go to "Tools" > "PsychoPy Shell" to open a command prompt window within PsychoPy.

In the terminal or command prompt window that opens, use the pip command to install the package you need. The command will look something like this:

pip install tobii-research

Now we should be ready to modify your experiment to include eyetracking.

Step 1: Import the necessary libraries

In any routine of your experiment, you can add a custom code component and add the following code to the "Before Experiment" tab.

## ! -----------IMPORT  --------------

import tobii_research as tr
from datetime import datetime
import csv
import pandas as pd
import re

## ! ----------- END IMPORT  --------------

Step 2: Add some necessary functions for the eyetracking

Continue adding the following code to the "Before Experiment" tab.

global_marker = -1

eyetracking_folder_path = 'data/eyetracking/'
if not os.path.exists(eyetracking_folder_path):
    os.makedirs(eyetracking_folder_path)
global_gaze_data = []

def gaze_data_callback(gaze_data):
    global global_gaze_data
    left_gaze_point_on_display_area = gaze_data['left_gaze_point_on_display_area']
    left_gaze_validity = gaze_data['left_gaze_point_validity']
    right_gaze_point_on_display_area = gaze_data['right_gaze_point_on_display_area']
    right_gaze_validity = gaze_data['right_gaze_point_validity']
    
    
    gaze_point = ((left_gaze_point_on_display_area[0] + right_gaze_point_on_display_area[0]) / 2,
                  (left_gaze_point_on_display_area[1] + right_gaze_point_on_display_area[1]) / 2)
    
    left_pupil_diameter, left_pupil_validity = gaze_data['left_pupil_diameter'], gaze_data['left_pupil_validity']
    right_pupil_diameter, right_pupil_validity = gaze_data['right_pupil_diameter'], gaze_data['right_pupil_validity']
    
    global_gaze_data.append((gaze_data['device_time_stamp'], gaze_data['system_time_stamp'],
                            gaze_point[0], gaze_point[1],
                            left_gaze_point_on_display_area[0], left_gaze_point_on_display_area[1],
                            right_gaze_point_on_display_area[0], right_gaze_point_on_display_area[1],
                            left_gaze_validity, right_gaze_validity,
                            left_pupil_diameter, right_pupil_diameter,
                            left_pupil_validity, right_pupil_validity,
                            global_marker))
    return True

def output_gaze_data(global_gaze_data, output_gaze_path):  
    with open(output_gaze_path ,'w',newline='') as out:
        csv_out = csv.writer(out)
        csv_out.writerow(['device_time_stamp', 'system_time_stamp',
                          'gaze_point_x', 'gaze_point_y',
                            'left_gaze_point_on_display_area_x', 'left_gaze_point_on_display_area_y',
                            'right_gaze_point_on_display_area_x', 'right_gaze_point_on_display_area_y',
                            'left_gaze_point_validity', 'right_gaze_point_validity',
                            'left_pupil_diameter', 'right_pupil_diameter',
                            'left_pupil_validity', 'right_pupil_validity',
                            'event'])
        for row in global_gaze_data:
            csv_out.writerow(row)
            
## ----------- END EYETRACKING --------------

These code are universal and can be used in any experiment.

The gaze_data_callback function is used to collect the gaze data. The output_gaze_data function is used to save the gaze data to a csv file.

Step 3: Connect to the eyetracker

Add the following code to the "Begin Experiment" tab of the same custom code component you just created.

eyetrackers = tr.find_all_eyetrackers()
if len(eyetrackers) == 0:
    print("No eyetrackers found")
else:
    print("Successfully found")
    eyetracker = eyetrackers[0]
    print("Using eyetracker:", eyetracker.model, eyetracker.serial_number)
eyetracker.subscribe_to(tr.EYETRACKER_GAZE_DATA, gaze_data_callback, as_dictionary=True)

Step 4: Add code for outputing eye tracking data

Add the following code to the "End Experiment" tab of the same custom code component you just created.

# Save eyetracking data
eyetracking_filename = filename.split('.')[0].split('/')[-1] + '_eyetracking.csv'
eyetracking_output_path = eyetracking_folder_path + eyetracking_filename
print('Saving eyetracking data to {}'.format(eyetracking_output_path))
output_gaze_data(global_gaze_data, eyetracking_output_path)

Step 5: Add functions for markers

Technically the eyetracking should be working now, but if you want to track when a specific event happens in the eye tracking data you collected, we can add 'markers' to the eyetracking data.

The core idea is to use a global variable to store the marker, and then add the marker to the gaze data when the event happens.

Add the following code to the "Before Experiment" tab of the custom code component.

Step 6: Marker query

First get a list of the events you want to track. For example, in the eBIP experiment, we want to know when a trial starts and ends, and when the audio/image starts playing in each trial. You can create a csv to store the markers. It should look similar to this:

block field item1 item2 instruction marker
perception Color marshmello baguette light 10
imagery Letter student summary up 20
... ... ... ... ... ...

It should be note that the items should be the refering as same items in the experiment. For example, in the the eBIP experiment, the items are the same as the image and audio files (although the file extension is removed, we can match them together by postprocessing).

Now we need a function to query the marker based on what is happening on the screen, for example when a perception trial containing marshmello.png and baguette.png starts, from the marker reference we can know the marker to the trial is 10.

Here is an example of how we query the marker in the eBIP experiment.

def marker_query(block, item1, item2, event):
    if block == 'vviq':
        item2 = str(item2)
        exp_marker = marker_reference[(marker_reference['item2'] == item2)]['marker'].values[0]
    else:
        pattern = '[\\\/](.*)\.'
        item1 = str(item1)
        item2 = str(item2)
        if re.search(pattern, item1):
            item1 = re.search(pattern, item1).group(1)
        if re.search(pattern, item2):
            item2 = re.search(pattern, item2).group(1)
        exp_marker = marker_reference[(marker_reference['block'] == block) & (marker_reference['item1'] == item1) & (marker_reference['item2'] == item2)]['marker'].values[0]
   
    if event == 'start':
        event_marker = exp_marker
    elif event == 'item1':
        event_marker = exp_marker + 1
    elif event == 'item2':
        event_marker = exp_marker + 2
    elif event == 'instruction':
        event_marker = exp_marker + 3
    elif event == 'vividness':
        event_marker = exp_marker + 4
    elif event == 'confidence':
        event_marker = exp_marker + 5   
    else:
        event_marker = exp_marker + 9

    return event_marker

Since there are multiple events we want to track in one trial (first audio, second audio, first image, second image, instruction, vividness, confidence, etc.), we need to add a number to the marker to distinguish them. So the marker in the marker reference is always ended with 0, and the following events are added by 1, 2, 3, 4, 5, 6, 7, 8, 9.

The code

pattern = '[\\\/](.*)\.'
item1 = str(item1)
item2 = str(item2)
if re.search(pattern, item1):
    item1 = re.search(pattern, item1).group(1)
if re.search(pattern, item2):
    item2 = re.search(pattern, item2).group(1)

is used to remove the file extension from the item name, so we can match the item name with the marker reference.

Step 7: Apply the marker functions and query to your experiment

Here is the final part where we put together what we have done so far.

With the marker_query function, we can get the marker at any point in the experiment. Then we can use the start_marker and end_marker functions to add the marker to the gaze data.

It's a bit complicated to explain how to do this in a general way, because it depends on the structure of your experiment. But here are two examples in the eBIP that may help you understand.

Objective1: Add a marker for the vviq block, which we only want to track the start and end of the trial.

First locate the vviq block in the experiment flow, and add a custom code component to the start of the routine. Then add the following code to the "Begin Routine" tab.

myCodeExecuted = False

So that the code will only be executed once in the routine.

Then add the following code to the "Each Frame" tab.

if text_senario.status == STARTED and not myCodeExecuted:
    vviq_event_marker = marker_query('vviq', senario, item, 'start')
    start_marker(vviq_event_marker)
    myCodeExecuted = True

Here text_senario is the text component name we set in psychoPy, and senario and item are the variables we set in the loop (column names of the vviq trials csv). What it does is when the text shows up, it will add a marker to the gaze data based on the senario and item variables.

Then add the following code to the "End Routine" tab.

end_marker()

To reset our marker once the routine ends.

Objective2: Add a marker for the imagery block, which we want to track the start and end of the trial, and the start of the audios.

Here we need to keep track of multiple events in one routine, but the idea is the same as before. We just need to call the start_marker multiple times and set different variables to track whether the code has been executed.

In the "Begin Routine" tab.

myCodeExecuted_trial = False
myCodeExecuted_word1 = False
myCodeExecuted_word2 = False
myCodeExecuted_instr = False

In the "Each Frame" tab.

if fixation.status == STARTED and not myCodeExecuted_trial:
    # ! Imagery Block event start
    imagery_current_marker = marker_query('imagery', word1, word2, 'start')
    start_marker(imagery_current_marker)
    myCodeExecuted_trial = True
    
if Mots1.status == STARTED and not myCodeExecuted_word1:
    # ! Imagery Block word1 start
    imagery_current_marker = marker_query('imagery', word1, word2, 'item1')
    start_marker(imagery_current_marker)
    myCodeExecuted_word1 = True
    
if Mots2.status == STARTED and not myCodeExecuted_word2:
    # ! Imagery Block word1 start
    imagery_current_marker = marker_query('imagery', word1, word2, 'item2')
    start_marker(imagery_current_marker)
    myCodeExecuted_word2 = True

if instrMots.status == STARTED and not myCodeExecuted_instr:
    # ! Imagery Block instruction start
    imagery_current_marker = marker_query('imagery', word1, word2, 'instruction')
    start_marker(imagery_current_marker)
    myCodeExecuted_instr = True

In the "End Routine" tab.

end_marker()
